Capsule ships as the `cpsl` Python package and includes the `capsule` CLI.

## Requirements

- Python `3.10+`
- a virtual environment or `uv` project
- a Capsule workspace if you want to `serve`, `deploy`, manage secrets, or create channels

## Install with uv

If you use `uv`, add Capsule to your project:

```bash
uv add cpsl
```

Or install it into a virtual environment directly:

```bash
uv venv
source .venv/bin/activate
uv pip install cpsl
```

## Install with pip

```bash
python -m venv .venv
source .venv/bin/activate
pip install cpsl
```

## Verify the CLI

```bash
capsule --help
```

You should see the main command groups:

```text
bootstrap
login
app
channel
deploy
secret
serve
fs
```

## Authenticate

Before you use hosted Capsule services, log in:

```bash
capsule login
```

This opens a browser, authenticates against your workspace, and stores the token locally so `serve`, `deploy`, `secret`, `channel`, and `fs` commands can talk to the gateway.

If you are targeting a local gateway, set the gateway URL first:

```bash
export CAPSULE_GATEWAY_URL=localhost:1980
capsule login
```

## Scaffold a project

To start from a template instead of a blank file:

```bash
capsule bootstrap my-agent-app
```

The bootstrap command can generate:

- `minimal` for a tiny message handler
- `chatbot` for a BAML-backed chat app
- `dashboard` for chat plus a custom React page

## What gets installed

The `cpsl` package includes both the Python SDK and the CLI:

- `cpsl.App`, `cpsl.Session`, `cpsl.Image`, `cpsl.Secret`, and the rest of the framework API
- the `capsule` CLI for `serve`, `deploy`, `login`, secrets, channels, and filesystems
