Installing Capsule is straightforward: install `cpsl`, make sure the `capsule` CLI is on your path, then log into your workspace.

One small thing is worth calling out early. `capsule serve` and `capsule deploy` are not just local helper scripts. They talk to the Capsule platform, so logging in is part of normal local setup, not something you wait to do later.

The `cpsl` package gives you both pieces you need:

- `cpsl`, the Python API you write your app with
- `capsule`, the CLI you use for `login`, `serve`, `deploy`, secrets, channels, and filesystems

## Requirements

- Python `3.10+`
- a virtual environment or `uv` project
- access to a Capsule workspace if you want to `serve`, `deploy`, manage secrets, create channels, or mount filesystems

## Install with uv

If you use `uv`, add Capsule to your project:

```bash
uv add cpsl
```

Or install it into a virtual environment directly:

```bash
uv venv
source .venv/bin/activate
uv pip install cpsl
```

## Install with pip

```bash
python -m venv .venv
source .venv/bin/activate
pip install cpsl
```

## Verify the CLI

```bash
capsule --help
```

You should see the main command groups:

```text
bootstrap
login
app
channel
deploy
fs
secret
serve
```

## Authenticate to your workspace

Before you do anything with `serve` or `deploy`, log in:

```bash
capsule login
```

This opens a browser, authenticates against your workspace, and stores the token locally so the CLI can talk to Capsule.

This is the step people often skip on the first pass. If `capsule serve` feels like it should work without authentication, that is a fair guess. In practice, it is booting Capsule's dev loop around your app definition, so workspace auth is part of the normal path.

If you are targeting a local gateway, set the gateway URL first:

```bash
export CAPSULE_GATEWAY_URL=localhost:1980
capsule login
```

## Scaffold a project

If you would rather start from a template than a blank `app.py`, use `bootstrap`:

```bash
capsule bootstrap my-agent-app
```

The bootstrap command can generate:

- `minimal` for a tiny message handler
- `chatbot` for a BAML-backed chat app
- `dashboard` for chat plus a custom React page

The `minimal` template is good if you want to understand the smallest possible app. `dashboard` is a better starting point if you already know you want chat plus a custom page.
