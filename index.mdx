It lets you:

- Build chat apps with persistent sessions history
- Deploy hosted UIs that users can sign into with batteries-included auth
- Run background tasks and schedules without a separate worker stack
- Add custom pages, dashboards, and workflow components
- Let signed-in users connect integrations like Gmail, GitHub, Slack, or AWS
- Store records in collections, mount filesystems, and use workspace secrets
- Connect external channels like Telegram, Slack, and WhatsApp to your app

You define the app in Python. Capsule handles the runtime, routing, hosted app shell, and deployment around it.

When you deploy, you get a fully self contained product. Each deployed Capsule app has its own authenticated users and access rules, so one app's user pool stays separate from another's.

Here is a complete, minimal example of an LLM-backed Capsule app:

```python
import cpsl
from openai import AsyncOpenAI

client = AsyncOpenAI()

app = cpsl.App(
    name="example-chat",
    image=cpsl.Image(python_packages=["openai"]),
    secrets=["OPENAI_API_KEY"],
)


@app.message()
async def handle(session: cpsl.Session, msg: cpsl.Message):
    response = await client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": "You are a concise operations assistant."},
            *session.chat_messages(msg),
        ],
    )
    await session.reply(response.choices[0].message.content or "No response.")
```

That's it. Save it as `app.py` and run:

```bash
capsule serve app:app
```

## How does it work?

Capsule takes your app definition, builds the runtime it needs, and runs it in Capsule's managed environment. The same app definition can expose chat, pages, tasks, data handlers, API routes, and channels.

`capsule serve` does this for local development. `capsule deploy` turns the same app into a hosted version. As the app grows, you keep adding to the same Python definition instead of moving into a separate worker system, page backend, or deployment config.

## Getting started

Developing with Capsule is simple because you do not need to assemble the runtime and product scaffolding yourself. Start with:

1. Install `cpsl` and verify the CLI with [Installation](/installation)
2. Run `capsule login` to authenticate to your workspace
3. Follow the [Quickstart](/quickstart) to build and run your first app

Then deploy it and start inviting users into the hosted app.
