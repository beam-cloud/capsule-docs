This is the shortest path from an empty project to a live Capsule app.

You will build one app with chat and a small page, run it locally, then deploy the same entry point.

## 1. Install Capsule

Install `cpsl`. The package includes the Python API and the `capsule` CLI.

```bash
uv add cpsl
```

or:

```bash
pip install cpsl
```

Verify the CLI:

```bash
capsule --help
```

## 2. Authenticate

```bash
capsule login
```

This authenticates your local machine as a builder. Your deployed app will still get its own app-scoped user sign-in flow.

## 3. Add the model dependency

```bash
uv add openai
```

or:

```bash
pip install openai
```

Make the key available locally:

```bash
export OPENAI_API_KEY=sk-...
```

Create the matching workspace secret for deploys:

```bash
capsule secret create OPENAI_API_KEY=sk-...
```

## 4. Create `app.py`

```python
import cpsl
from openai import AsyncOpenAI

client = AsyncOpenAI()

app = cpsl.App(
    name="hello-capsule",
    image=cpsl.Image(python_packages=["openai"]),
    secrets=["OPENAI_API_KEY"],
)


@app.page("Status", icon="layout-dashboard")
def status_page():
    return cpsl.ui.Page(
        [
            cpsl.ui.Text("Hello from Capsule", style="heading"),
            cpsl.ui.Text(
                "Chat, pages, and deploys all come from the same app definition.",
                style="muted",
            ),
        ]
    )


@app.message()
async def handle(session: cpsl.Session, msg: cpsl.Message):
    response = await client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {
                "role": "system",
                "content": (
                    "You are a concise product assistant. Help the user think "
                    "through agentic app ideas, channels, pages, tasks, and "
                    "integrations."
                ),
            },
            *session.chat_messages(msg),
        ],
    )

    text = response.choices[0].message.content or "I could not generate a reply."
    session.data["last_response"] = text
    await session.reply(text)
```

This file has three pieces:

- `App(...)` defines the runtime and deploy config
- `@app.page(...)` adds a sidebar page
- `@app.message()` registers the chat handler

The same app definition owns all three.

## 5. Serve locally

```bash
capsule serve app:app
```

`app:app` means "load the symbol `app` from `app.py`."

Open the preview URL from the command output. You should see:

- chat, powered by `@app.message()`
- a `Status` page, powered by `@app.page(...)`

Try a product-shaped prompt:

- `I want to build a Telegram concierge for travel agents. What should the app include?`
- `How would you structure the state, pages, tasks, and integrations?`

## 6. Deploy

When the app works locally:

```bash
capsule deploy app:app
```

Deploy packages the same app definition and returns a hosted URL. Users sign into that hosted app; the signed-in user is available inside handlers as `session.user`.
