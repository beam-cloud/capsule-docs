description: "Build and run a minimal Capsule app in a few minutes."

This guide is intentionally small. The goal is not to show every Capsule feature. The goal is to get one app running end to end so the rest of the docs have something concrete to build on.

By the end you will have:

- a local Capsule app
- a working message handler
- one small piece of session state
- a successful deploy

## 1. Create `app.py`

```python
import cpsl

app = cpsl.App(
    name="hello-capsule",
    image=cpsl.Image(),
)


@app.message()
async def handle(session: cpsl.Session, msg: cpsl.Message):
    await session.reply(
        f"Hello {session.user.email or 'there'}.\n\n"
        f"You said: {msg.text}"
    )
```

This is the functional style, which is the easiest place to start.

- `App(...)` defines the runtime and deploy config
- `image=cpsl.Image()` says the app should run in a default Python environment
- `@app.message()` registers the chat handler
- `session.reply()` sends a message back to the user

If you are wondering why `image=` is required here: in functional apps, `App(...)` needs to know the runtime environment up front. In class-based apps, that same information lives on `@app.cls(...)` instead.

## 2. Start the local dev loop

```bash
capsule serve app:app
```

`app:app` means:

- module: `app.py`
- symbol: `app`

`capsule serve` uploads the project, boots the runtime, and hot-reloads when you edit watched files such as Python, TSX, CSS, JSON, and Markdown.

If this is your first time using Capsule on this machine, you may need to run `capsule login` first.

## 3. Talk to the app

Open the local preview from the `capsule serve` output and send a message in chat.

Try editing the handler and saving the file:

```python
@app.message()
async def handle(session: cpsl.Session, msg: cpsl.Message):
    session.data["last_message"] = msg.text
    await session.reply(
        f"Last message saved for session {session.id[:8]}.\n"
        f"Current text: {msg.text}"
    )
```

Now the app is doing something slightly more useful:

- `session.data` stores per-conversation state
- `session.id` gives you a stable session identifier
- saving the file should cause the local preview to pick up the change

At this point you have already seen the two most common Capsule building blocks: a message handler and session state.

If you want to inspect the current conversation state more directly, try this version:

```python
@app.message()
async def handle(session: cpsl.Session, msg: cpsl.Message):
    session.data.setdefault("messages", []).append(msg.text)

    await session.reply(
        "Stored messages:\n" +
        "\n".join(f"- {text}" for text in session.data["messages"])
    )
```

## 4. Deploy it

When the local version looks good:

```bash
capsule deploy app:app
```

Deploy packages the same app definition you just served locally and sends it to Capsule. The deploy output includes the hosted URL for the app.

In other words, you are not maintaining a separate deployment config somewhere else. The Python code is the source of truth.

## 5. Add the next feature

From here, most apps branch into one or more of these directions:

- collections for durable data and tables
- data handlers for computed JSON
- DSL or React pages for structured UI
- tasks and schedules for asynchronous work
- integrations and secrets for external systems

A common next step is to add a page so the app is not chat-only anymore:

```python
@app.page("Status", icon="layout-dashboard")
def status_page():
    return cpsl.ui.Page(
        [
            cpsl.ui.Text("Hello from Capsule", style="heading"),
            cpsl.ui.Text("This is your first page.", style="muted"),
        ]
    )
```
